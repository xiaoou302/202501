# Zenion 游戏机制更新说明

## 新游戏流程

### 第一阶段：闪烁收集（Glowing Phase）
1. 游戏开始时，所有文字以普通颜色显示（黑色）
2. 关键词随机顺序开始闪烁（蓝色发光）
3. 每个关键词闪烁3次，每次持续2秒
4. 玩家需要在闪烁时点击收集关键词
5. 错过闪烁后，该关键词无法再点击
6. 收集的关键词显示在右侧面板

### 第二阶段：重建填空（Reconstruction Phase）
1. 所有关键词闪烁完毕后，自动进入重建阶段
2. 文章中的所有关键词位置变成空白框
3. 玩家可以：
   - **拖拽**：从右侧收集面板拖拽词语到空白处
   - **输入**：如果未收集到某些词，可以直接输入
4. 填满所有空白后，点击"提交修复"按钮
5. 系统验证每个位置的词语是否正确

### 验证规则
- 必须**位置正确**：词语必须在原来的位置
- 必须**内容正确**：词语必须与原文一致
- 100%正确 = 3星
- 80%以上正确 = 2星
- 60%以上正确 = 1星（通关）
- 60%以下 = 失败

## 技术实现要点

### 1. 文章简短化
- 每章减少到2-3句话
- 确保在屏幕上完整显示，无需滚动
- 关键词数量：12-16个

### 2. 闪烁系统
- 使用Timer控制闪烁序列
- 随机打乱关键词顺序
- 每个词闪烁3次：出现在队列中3次
- 闪烁间隔：2秒发光 + 0.5秒间隙

### 3. 拖拽实现
- 使用Draggable和DragTarget
- 从collection panel拖到text blanks
- 支持撤销拖拽

### 4. 输入实现
- 空白处提供TextField
- 可以手动输入未收集的词
- 自动验证

## 用户体验改进

1. **视觉反馈更清晰**
   - 普通文字：黑色
   - 闪烁关键词：蓝色发光
   - 已收集：绿色标记
   - 空白：橙色边框

2. **操作更直观**
   - 闪烁时点击收集
   - 拖拽填空
   - 手动输入补充

3. **难度平衡**
   - 每词3次机会
   - 可通过上下文推理
   - 60%通过线足够宽松
